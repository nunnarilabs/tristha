# Test Strategy — Template (Cowork should follow this structure)

1. **Scope** — features in/out of scope
2. **Test approach** — levels (SIT/UAT), types (functional, negative, boundary, security, performance)
3. **Entry & exit criteria**
4. **Key risks & mitigations**
5. **Test data approach** — synthetic, PII-safe
6. **Coverage summary** — requirements → planned scenarios
7. **Open questions / assumptions**
